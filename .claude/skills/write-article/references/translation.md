# 翻译与链接提取

适用于：用户给英文文章全文，或给一个 URL。

## 链接提取（输入是 URL 时）

1. 用 WebFetch 获取页面，prompt 要求返回文章完整正文、标题、作者、发表日期
2. 内容明显被截断时（正文戛然而止、缺结尾），换一个 prompt 再取一次，或分段提取
3. 提取后判断语言：英文走下面的翻译流程；中文走 `formatting.md` 的排版入库流程（同样要附出处，见下）
4. 提取失败（付费墙、需登录）时如实告诉用户，请他们把全文粘贴过来，不要凭片段硬翻

## 翻译原则

目标是让中文读者读起来像一篇本来就用中文写的文章，而不是逐句对照的译文：

- **意译优先**：拆开英文长句，按中文习惯重组语序；定语从句、被动语态不要硬搬
- **术语**：心理学术语首次出现时附英文原文，如"心流（flow）"；之后只用中文。译名优先沿用通行译法（积极心理学、正念、自我决定理论）
- **人名书名**：知名学者用通行中译名（如米哈里·契克森米哈赖）；不确定有无通行译名时保留英文原名。书名查证有无中译本，有则用中译本书名
- **文化语境**：英文俚语、美国特有的文化引用，换成中文读者能懂的表达或加一句说明
- **不增删观点**：可以调整表达，不能添加原文没有的论断，也不能删掉不合自己口味的段落
- 正文排版遵守 `formatting.md` 的中文排版规则

## 出处标注（必须）

翻译和转载的文章**直接发布**（不加 draft），但必须在正文末尾标注出处，固定格式：

```markdown
---

> 本文翻译自 [原文标题](原文链接)，作者 作者名，发表于 原发表日期或站点名。中文版由本站翻译，如有偏差以原文为准。
```

- 用户只粘贴了全文没给链接时，向用户要原文链接和作者名；要不到就写"本文翻译自 作者名 的文章《原文标题》"
- 中文转载（非翻译）时格式改为："> 本文转载自 [原文标题](原文链接)，作者 作者名。"

## frontmatter 注意点

- `title` 用译后的中文标题，可以为了达意而不直译
- `summary` 用中文重新概括，不翻译原文的 meta description
- `pubDate` 用入库当天日期（不是原文发表日期，原文日期写在出处标注里）
- `tags` 按文章主题选，复用站内现有 tag
