# 排版规范与 frontmatter 模板

适用于：原创写作、排版入库，以及其他模式生成正文时的通用规范。

## posts frontmatter 模板

```yaml
---
title: 文章标题（中文，不带书名号，除非标题本身引用书名）
pubDate: 2026-06-11          # 当天日期，date +%Y-%m-%d
summary: 一句话摘要，60 字以内，会显示在列表页和搜索结果里。
tags: [心理学]               # 至少一个，优先复用现有 tag
# 以下仅访谈类文章需要：
# interviewee: 受访者姓名
# intervieweeBio: 一句话介绍受访者背景。
# cover: 可选，posts 一般不配封面
---
```

## 中文排版规则

写正文（包括翻译产出的正文）时遵守：

- **标点**：正文用全角标点（，。：；？！），不混用半角逗号句号
- **中英文之间加空格**：如"积极心理学（positive psychology）的研究"中，中英文、中文与数字之间留一个半角空格："每天 15 分钟"
- **引号**：强调和引用术语用直角引号「」；多层嵌套用『』；不用弯引号""
- **书名**：用书名号《》，首次出现可附原文名："《心流》（Flow）"
- **引文**：整段引用用 Markdown blockquote（`>`），与现有文章一致
- **小标题**：正文超过 600 字时用 `##` 分节，小标题简短、不带序号
- **段落**：每段聚焦一个意思，3-5 句为宜；不写超长段落

## 排版入库模式的额外要求

用户给的是已经写好的文章，你的职责是排版而不是改写：

- **不改动作者的观点、论证和措辞风格**，只修正标点、空格、引号、错别字等机械问题
- 按上面规则统一标点和空格
- 替作者拟 `summary` 和 `tags`（这是 frontmatter 必填项，作者通常没写）
- 如果文章没有标题，根据内容拟一个，并在交付时告诉用户标题是你拟的
- 改动较多时（比如统一了几十处标点），在交付时简要说明改了哪几类问题

## 原创写作模式的额外要求

- 先列出文章要回答的核心问题，再动笔；不写"幸福很重要，让我们一起追寻幸福"式的空话
- 观点尽量落到可操作的层面（怎么做、什么场景下做），而不是停留在"要心怀感恩"
- 引用心理学研究时给出研究者或著作名，宁可少引也不编造
- 篇幅看素材定，一般 800-1500 字；素材撑不起长文时宁短勿水
