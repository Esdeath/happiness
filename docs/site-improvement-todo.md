# 网站改进 TODO

基于当前站点状态整理：技术基础已经较完整，优先补内容厚度、专题组织、可信度和回访理由。

## P0：先提升完成度

- [ ] 增加成体系的内容栏目
  - 建议栏目：读书笔记、幸福实验、人物访谈、工作与幸福、关系与幸福。
  - 目标：让读者不只读单篇文章，而是能沿着一个主题继续读下去。

- [ ] 优化首页定位说明
  - 在首页加入一段克制的站点说明，明确这里提供文章、访谈、书单和幸福相关思考。
  - 增加“从这里开始”入口，推荐 3 篇代表作或 3 个核心主题。

- [ ] 补充作者可信度
  - 将 `src/config.ts` 中的作者从站名兜底改成真实作者或明确的编辑身份。
  - 在关于页补充作者简介、写作原则、资料来源标准和联系方式。

- [ ] 增加专题页或起步路径
  - 建议专题：目标达成之后为什么不快乐、罗素与幸福、积极心理学入门书单。
  - 专题页应人工策划内容顺序，而不是只依赖标签聚合。

## P1：提高回访和互动

- [ ] 增加普通读者友好的订阅入口
  - RSS 已有，但可以补一个更明显的订阅页面或按钮。
  - 可选渠道：邮件、公众号、即刻、Telegram，或先做 `/subscribe` 页面。

- [ ] 开通评论或反馈
  - 当前已有 Giscus 组件，但 `src/config.ts` 中 `GISCUS.repo` 为空，所以评论区不会渲染。
  - 配好 GitHub Discussions 和 Giscus 后，文章和书籍详情页会自动显示评论区。

- [ ] 补实书单内容
  - 替换仍带“书评示例”性质的内容。
  - 每本书建议补齐：推荐理由、适合谁、不适合谁、相关读书笔记、购买链接利益披露。

## P2：增强阅读体验和信任

- [ ] 增加长文阅读增强
  - 可选功能：目录、阅读时间、上一篇/下一篇、脚注样式、引用样式。
  - 重点用于较长文章，不必先做成复杂系统。

- [ ] 补充基础信任页
  - 建议新增：隐私政策、转载说明、联盟链接说明。
  - 备案号已有，后续可以把这些页面统一放在页脚。

- [ ] 增加轻量数据分析
  - 可选方案：Cloudflare Web Analytics 或其他无 Cookie 分析工具。
  - 目标：了解哪些主题有人读，再决定下一批内容方向。

## 建议执行顺序

1. 首页“从这里开始”模块。
2. 真实作者与写作原则。
3. 3 个专题入口。
4. 补实书单正文。
5. 订阅入口和评论配置。
