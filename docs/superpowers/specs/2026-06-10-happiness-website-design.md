# 「追寻幸福」网站设计文档

日期：2026-06-10
状态：已与站主确认设计，待审阅书面规格

## 1. 背景与目标

做一个以"追寻幸福"为主题的中文内容网站，包含三类内容：

1. 幸福主题的原创文章
2. 访谈文章
3. 推荐书单（带购买链接，预留佣金链接位）

站主独自运营，用 Markdown 写作，`git push` 即发布。目标是以零成本、零运维的方式先把内容站跑起来。

## 2. 受众与约束

- **受众**：中文读者为主，多数从移动端（微信分享）访问
- **部署**：Cloudflare Pages（免费、零运维；后续如迁移国内服务器，静态产物可直接平移）
- **佣金**：暂无联盟账号（京东联盟/当当联盟/淘宝客均未申请）。设计上每本书预留购买链接列表，拿到佣金链接后直接替换 URL 字段，不需要改代码
- **预算**：零成本（托管、评论、搜索均使用免费方案）

## 3. 技术选型

**Astro 5 静态站点**。选型理由：

- 专为内容站设计，Content Collections 提供带 schema 校验的 Markdown 内容管理
- 默认输出纯静态 HTML、近零 JavaScript，性能与 SEO 最佳
- 所需四项功能（标签、搜索、RSS、评论）在 Astro 生态均有成熟零成本方案

已考虑并排除的替代方案：

- **Next.js + MDX**：能力上限高，但对纯内容站复杂度过剩，搜索/RSS 需自建
- **Hugo/Hexo**：现成主题上手快，但访谈、书单等自定义内容结构的模板定制体验差

## 4. 内容模型

两个 Content Collection，文章与访谈合并为一个集合：

```
src/content/
├── posts/    # 文章 + 访谈
└── books/    # 推荐书籍
```

### posts 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 标题 |
| pubDate | date | 是 | 发布日期 |
| summary | string | 是 | 摘要，用于列表页和 SEO description |
| tags | string[] | 是 | 标签，如 心理学/哲学/生活方式；访谈统一加"访谈"标签 |
| cover | image | 否 | 封面图 |
| interviewee | string | 否 | 受访者姓名；填写则按访谈渲染 |
| intervieweeBio | string | 否 | 受访者简介，渲染在文章顶部介绍块 |
| draft | boolean | 否 | true 时构建跳过，默认 false |

访谈正文约定：用 Markdown 引用块或加粗区分问与答。

### books 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 书名 |
| author | string | 是 | 作者 |
| cover | image | 是 | 封面图 |
| oneLiner | string | 是 | 一句话推荐语 |
| tags | string[] | 是 | 标签 |
| purchaseLinks | {platform, url}[] | 是 | 购买链接列表，如 `[{platform: "京东", url: "..."}]`，可为空数组 |
| draft | boolean | 否 | 同上 |

正文为完整推荐理由/书评。

## 5. 页面结构

```
/                  首页：站点理念一句话 + 最新文章 + 精选书籍
/posts             文章列表（含访谈），时间倒序，支持按标签筛选
/posts/[slug]      文章详情；含受访者字段时渲染介绍块
/books             书单页：卡片列表（封面 + 一句话推荐语）
/books/[slug]      书籍详情：封面、作者、书评 + 醒目"去购买"按钮组
/tags/[tag]        标签聚合页：该标签下的所有文章和书
/about             关于页
/rss.xml           RSS 订阅源
/404               自定义 404 页
```

## 6. 功能实现

- **标签/分类**：frontmatter `tags` 数组，构建时静态生成标签页
- **站内搜索**：Pagefind，构建后生成静态全文索引（中文支持好），搜索框在导航栏
- **RSS**：`@astrojs/rss` 官方插件
- **评论**：giscus，挂在 posts 与 books 详情页底部，数据存于 GitHub Discussions
- **购买链接**：详情页按钮组，外链统一 `rel="nofollow sponsored"`，新窗口打开

## 7. 视觉方向

基调：**温暖、安静、留白充裕**——"安静的读书空间"而非"资讯门户"。

- 米白/暖白背景 + 一个温暖强调色（暖橘或赭黄）
- 标题用衬线字体（人文感），正文用无衬线（屏幕可读性）
- 中文排版：行高约 1.8、合理段间距、标点悬挂
- 移动端优先

## 8. 部署

- 代码托管 GitHub，连接 **Cloudflare Pages**，push 即自动构建发布
- 域名后续绑定

## 9. 错误处理

- 内容 schema 校验：字段错误构建即失败并定位到文件，不带病上线
- 自定义 404 页面
- giscus 加载失败不影响正文（评论区独立降级）
- 购买链接为纯外站跳转，无运行时依赖

## 10. 测试策略

- CI 跑 `astro build`，schema 校验兜底
- 少量单元测试覆盖工具函数（标签聚合、RSS 生成）
- 不引入 E2E 框架（维护成本超过收益）

## 11. 非目标（本版明确不做）

- 可视化后台/CMS（内容即 Markdown 文件）
- 联盟 API 自动化集成（手动贴链接）
- 用户系统、登录、收藏
- 多语言/英文版
- 国内服务器部署与 ICP 备案（保留未来迁移可能）
