# 幸福视角书籍总结 Skill 设计

## 目标

修改项目内的 `write-article` skill，让书籍总结围绕本站主题选材。除作者直接讨论的幸福之外，还应提炼与幸福密切相关的意义感、关系、情绪、自由与自主、工作与投入、身心健康及生活实践。

产出仍包括书单条目、读书笔记和封面。文件位置、frontmatter、封面来源、构建与发布流程不变。

## 修改范围

修改两个文件：

- `.agents/skills/write-article/SKILL.md`：把书籍模式描述为面向幸福及相关主题的定向总结。
- `.agents/skills/write-article/references/book-summary.md`：加入选材框架、写作结构和交付前检查。

不修改站点内容、内容 schema、页面组件或其他写作模式。

## 选材框架

总结前先识别书中实际涉及的主题：

- 幸福、满足与痛苦
- 意义感、价值与人生方向
- 关系、爱、归属与社会连接
- 情绪、心理韧性与自我接纳
- 自由、自主与选择
- 工作、投入、成长与成就
- 身体健康、休息与生活节奏
- 能落实到日常生活的做法

只写有可靠依据的主题，不要求覆盖完整清单。书中没有直接讨论幸福时，可以从本站视角说明某一观点为何影响生活质量，但要明确区分作者原意与本站的关联分析。

## 输出要求

书单条目的 `oneLiner` 和推荐语应说明本书能帮助读者理解或改善幸福生活的哪个方面。

读书笔记开头先回答本书与幸福生活的关系。全书主旨只作简要交代，正文篇幅优先留给幸福相关主题。每个重点观点包含三项信息：作者主张、作者使用的理由或材料，以及该观点与幸福生活的关系。

实践部分只提炼书中能支持的建议，并写清使用场景、行动方式和适用边界。不能把转述扩写成作者未提出的处方。结尾说明适合哪些读者，并提示研究年代、因果限制、个人处境或专业支持等必要边界。

## 验收标准

交付前逐项检查：

- 标题、摘要、`oneLiner` 和正文共享一条清楚的幸福主题主线。
- 文章没有平均复述所有章节，主要篇幅用于幸福及相关主题。
- 观点能够区分作者原意、书中依据和本站分析。
- 没有虚构章节、引文、页码、研究结论或实践建议。
- 实践建议具体，并包含适用条件或限制。
- 原有文件、封面、构建、提交和发布规则仍然有效。

## 验证

使用 skill 校验脚本检查目录和 frontmatter；再检查 diff，确认修改只涉及设计说明和上述两个 skill 文件。由于本次不修改 Astro 内容或代码，不要求运行站点构建。
